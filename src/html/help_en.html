About Suika2 Kirara

Suika2 Kirara is a graphical implementation of the Suika2 Visual Novel Development Engine. This application allows developers to create Suika2Script scenario files and structures from within a cohesive environment and combines a number of Suika2 features with Suika2 Pro for Creators' export functionality. This is an early version of the application and documentation, please report any strange behaviour to the contact information noted at the bottom of this page to let us know.

What can Kirara do at the moment?

Kirara can...
  • Facilitate simple, timeline-based scenario editing.
  • Facilitate graphical Suika2Script editing for all currently supported commands.

What can't Kirara do at the moment?

Kirara can't...
  • Facilitate graphical GUI definition editing.
    • A GUI Library is planned as an alternative.
  • Facilitate graphical Suika2Script editing for all currently supported commands.
    • We will introduce the structured editing of branches (@choose, @if, and @goto).
  • Facilitate the use of the multilingual prefix feature.
    • This feature is currently under consideration.

System Requirements

Most modern Windows and Mac systems are capable of running Suika2 Kirara. More information regarding system requirements will be compiled when it is deemed appropriate.

Commands and Usage

While Kirara does not rely on keyboard-entered commands like Suika2 scenario files, it is helpful for a user to understand what each does. Like Suika2, Kirara uses Suika2Script as its main scenario blocks. For more information regarding Suika2Script, please see the corresponding Suika2 Wiki article.

What is Suika2Script?

Suika2Script is the markup language used in the creation of Suika2 script files (often referred to as scenario files in a general sense). Being such a versatile markup, the user only needs to learn a few concepts before they are able to utilise the entire feature-set from displaying messages, images, playing sounds, music, and even building menus.

Suika2Script is an actively developed markup, therefore it is subject to changes alongside Suika2 (and sister applications).

What are Commands?

If Suika2Script is Suika2's language, then commands are the individual words and sentences that make up that language – they may seem out of place on their own, but when paired with other commands, they are able to tell brilliant stories of varying levels of intricacy.

Command List

Command blocks are located on the left-hand side of the screen.

Click to Expand the Command List

Message

The message command block is used to draw a message within the message box.
This command block supports NVL markup.

Line

The line command block is used to draw a message within the message box and a name to the namebox.
This command supports an optional audio file (most often for character voices).
This command block supports NVL markup.

Options

The options command block is used to present a series of options (a maximum of 8) to the player.
These options can branch to differenct labels.

Stage Change

The stage change command block is used to change the background and characters at the same time.

Character Move

The character move command block is used to move a character.

Sound Volume

The sound volume command block is used to set game volume.

Bookmark

The bookmark command block acts as a jumpable marker or scenario separator.
The equivalent written name is LABEL.

Jump to a Bookmark

The jump to a bookmark command block is used to jump to bookmarks.
The equivalent written name is @goto.

Set a Flag

The set a flag command block sets a variable.
This command supports variable operations such as +, -, x, and ÷ (or /).

Jump by a Flag

The jump by a flag command block jumps to a bookmark based on conditions set by a flag.

Chapter Title

The chapter title command block sets the chapter (reflected in the game windows titlebar).

Wait for a Click

The wait for a click command block pauses the game until the user clicks.

Timed Wait

The timed wait command block pauses the game for a specified number of seconds then resumes play.

Shake the Screen

The shake the screen command block shakes the screen for a specified number of seconds.

Allow Skip

The allow skip command block either allows or denies the user the ability to skip through command sections.

Advanced Script

The advanced script block allows the use of Watermelon Script files (used in script-calls).

Comment

The comment command block is not visable during play, and aids in scenario structure and notes.
